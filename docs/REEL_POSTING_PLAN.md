# Reel posting plan — turning the Bible Chat teardown into a calendar

_Companion to `docs/MARKETING_PLAN.md`. Source insight: the Bible Chat teardown (HANDOFF.md,
2026-07-04) — we can't out-spend a paid-acquisition volume business, so our lever is **organic
short-form with a weekly register rotation**. Direct evidence from our own accounts: a branded
reel got **147 IG views vs 0–22** for website screen-recordings. Format beats follower count;
a small account is fine._

We have **15 finished reels** (`tools/reel/specs/*.json`), all sourced from orthodoxy-gated
answer pages. This plan schedules them.

---

## The 6 operating rules (do these every time)

1. **Post the reel, never a screen-recording or screenshot.** This is the single biggest lever
   we have evidence for (147 vs 0–22).
2. **Upload the file natively to each platform** — TikTok, IG Reels, YouTube Shorts separately.
   Do **not** cross-post a link or a watermarked re-share; the algorithms suppress both.
3. **Add a trending sound** (low, under the voiceover) in the editor before posting. The reel is
   silent-by-design; the sound is a discovery signal, not the content.
4. **Add a voiceover** in one pass (CapCut / Canva / InShot) using the spec's `voiceover` field —
   captions carry the meaning, voice adds retention.
5. **Post ~5×/week** (Mon–Fri below). Consistency > volume; daily is the ceiling, not the floor.
6. **Re-pin the best-performing reel** on the TikTok profile, and save it as an IG "featured"
   Reel, so the profile's first impression is the winner.

**Register rotation:** never post two of the same *emotional register* back-to-back. The five
registers below are labeled on every calendar slot so the feed feels varied, not preachy.

---

## The editor pass (once per reel, before it enters the queue)

For each spec: open the MP4 in `tools/reel/output/`, drop in a **voiceover** (read the `voiceover`
field) and a **quiet trending sound**, export watermark-free. Keep the on-screen captions as-is.
~5 minutes each. Do a batch of 5 at a time so a week is always queued ahead.

## Platform-specific notes

- **TikTok** — highest organic reach for a small account; treat as primary. Post 12–6pm or 7–10pm.
  Re-pin the week's best. Reply to comments *with a video* when a question is good (free content).
- **Instagram Reels** — native upload, cover frame = the hook text, add 3–5 hashtags in the caption
  and the rest in a first comment. Cross-post the pastoral one to Stories.
- **YouTube Shorts** — same file; put the question in the title verbatim (Shorts is search-driven,
  so titles double as SEO). This is where the reels compound with our answer-page SEO.

## What to measure

The reels drive to `apologiadaily.com` → the matching `/answers/` page → free AI tool → account.
Right now we **cannot see** that funnel because PostHog is missing the events. Before/alongside
posting, wire (see `docs/MARKETING_PLAN.md`):
- `reel_link_click{slug}` — did a reel drive a click
- `answers_page_viewed{slug}` — did they land on the page
- `signup_completed` — did they convert

Until those exist, track platform-native numbers weekly per reel: **views, watch-through %, saves,
comments, profile taps**. Watch-through % and saves matter more than raw views. Kill nothing after
one post; a reel can catch on the 2nd or 3rd re-share.

## Week 4+ — recycle winners, feed the machine

- **Re-post the top 3** performers from weeks 1–3, each with a *new hook line / cover text* (same
  video) — winners keep winning and reach new viewers.
- **Cut square (1:1) + wide (16:9)** versions of the top 3 (`gen_reel.py --aspect square|wide`) for
  the feed / YouTube.
- **Draft the next batch** from the ~46 uncovered `/answers/` pages (resurrection + Bible-reliability
  clusters are the best SEO targets), keeping the register mix balanced.
- Fold the winners' topics into the **answer-page SEO** priority — a reel that pops signals a
  question with demand.
